Ops Runbook

Backup, recovery, and incident response playbook for LocalMarketNE.
Ops Health

1) Backups

  • Database: enable managed backups (daily) and confirm retention. Take a manual snapshot before major deploys.
  • Media/uploads: {% if use_s3 %}S3 is enabled. Recommend bucket versioning + lifecycle rules.{% else %}Local media is enabled. Ensure the host volume is backed up or migrate to S3 before launch.{% endif %}
  • Config: store environment variables securely (Render env / secrets manager). Never commit secrets.
Tip: run python manage.py ops_backup_report for a config/status snapshot.

2) Restore Procedure (High-level)

  1. Put site in maintenance mode (Ops) or temporarily disable checkout.
  2. Restore database snapshot from provider (Render/PG host) into a clean instance.
  3. Restore media (if applicable) to the expected bucket/path.
  4. Run migrations: python manage.py migrate
  5. Run smoke checks: /healthz/, /ops/health/, and a small test checkout in Stripe test mode.

3) Webhook & Payment Incidents

  • If Stripe shows payment succeeded but the order is not marked paid: check webhook logs and re-send the event (Stripe dashboard).
  • Ops should never "edit totals" directly. Prefer idempotent service functions that log AuditLog entries.
  • Verify Stripe keys present: {{ stripe_configured|yesno:"Yes,No" }}.
Pre-deploy gate: python manage.py rc_check --checks --db and python manage.py money_loop_check.

4) Email Incidents

  • Backend: {{ email_backend|default:"(not set)" }}
  • From: {{ default_from_email|default:"(not set)" }}
  • If verification resend breaks, verify the POST alias route exists: /accounts/verify-email/resend/.
